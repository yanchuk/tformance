## Overview

Technology detection categorizes PR files into: `frontend`, `backend`, `javascript`, `test`, `docs`, `config`, `other`.

## Categories

### Frontend
Extensions: `.jsx`, `.tsx`, `.vue`, `.svelte`, `.dart`, `.astro`, `.razor`, `.mdx`, `.css`, `.scss`, `.sass`, `.less`, `.html`, `.ejs`, `.hbs`, `.pug`

Path patterns:
- `/components/`, `/hooks/`, `/contexts/`, `/composables/`
- `/pages/`, `/views/`, `/layouts/`, `/screens/`
- `/frontend/`, `/client/`, `/web/`, `/ui/`
- `/apps/web/`, `/packages/ui/`

### Backend
Extensions: `.py`, `.php`, `.rb`, `.java`, `.kt`, `.scala`, `.cs`, `.go`, `.rs`, `.c`, `.cpp`, `.ex`, `.swift`, `.sh`, `.sql`

Path patterns:
- `/api/`, `/controllers/`, `/services/`, `/handlers/`
- `/models/`, `/repositories/`, `/migrations/`
- `/backend/`, `/server/`
- Framework-specific: `/app/Http/` (Laravel), `/lib/web/` (Phoenix)

### JavaScript (Ambiguous)
Extensions: `.js`, `.ts`, `.mjs`, `.cjs` when path doesn't match frontend/backend patterns

### Test
Patterns:
- Directories: `/tests/`, `/test/`, `/__tests__/`
- Files: `test_*.py`, `*.test.js`, `*.spec.ts`, `*_test.go`

### Documentation
Extensions: `.md`, `.rst`, `.txt`, `.adoc`, `.org`, `.tex`

### Configuration
Extensions: `.json`, `.yaml`, `.yml`, `.toml`, `.ini`, `.env`, `.lock`
Files: `Dockerfile`, `Makefile`, `.gitignore`, `.prettierrc`

### Other
Anything not matching above categories

---

## Path Priority (Tiered System)

1. **Test files** - Highest priority, any extension
2. **Backend exceptions** - `/pages/api/`, `/app/api/` (Next.js API routes)
3. **Frontend patterns** - `/components/`, `/hooks/`, etc.
4. **Backend patterns** - `/api/`, `/controllers/`, etc.
5. **Fallback** - "javascript" for ambiguous `.js`/`.ts`
