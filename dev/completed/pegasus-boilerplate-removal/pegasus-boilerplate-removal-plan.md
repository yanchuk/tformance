# Plan: Remove Pegasus Boilerplate Code

**Last Updated: 2025-12-28**

## Executive Summary

Remove unused SaaS Pegasus framework boilerplate code from the tformance codebase. This code was generated by the Pegasus Django SaaS boilerplate and serves only as tutorial/demo examples with zero dependencies in the actual application.

## Current State Analysis

The codebase contains Pegasus boilerplate in multiple locations:

| Component | Location | Files | Purpose |
|-----------|----------|-------|---------|
| Example Apps | `pegasus/apps/` | 23 | Demo forms, employees CRUD |
| Teams Example | `apps/teams_example/` | 10 | Example team-scoped model |
| Templates | `templates/pegasus/` | 26 | Demo HTML templates |
| Templates | `templates/teams_example/` | 1 | Player list template |
| JavaScript | `assets/javascript/pegasus/` | ~5 | React/Vue examples |
| Styles | `assets/styles/pegasus/` | 1 | Example CSS |
| Images | `static/images/pegasus/` | 14 | Demo images |
| Config | `pegasus-config.yaml` | 1 | Framework config |

### Integration Points (to be removed)

1. **settings.py**: `PEGASUS_APPS` list and `teams_example` in `PROJECT_APPS`
2. **urls.py**: Three URL patterns routing to example apps
3. **Celery**: `test-celerybeat` scheduled task

## Proposed Future State

- Clean codebase with only production application code
- No demo/tutorial routes accessible
- Reduced code surface area for maintenance
- Simplified settings and URL configuration

## Implementation Phases

### Phase 1: Configuration Updates (Effort: S)
Update Django settings and URL configuration to remove references.

### Phase 2: Directory Removal (Effort: S)
Remove all boilerplate directories and files.

### Phase 3: Verification (Effort: S)
Run tests and verify application functionality.
