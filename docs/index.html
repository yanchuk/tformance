Methodology

Data Collection

  • Source: GitHub GraphQL API with authenticated access
  • Projects: 25 open source projects (Vercel, PostHog, Cal.com, Supabase, etc.)
  • Date Range: January 1, 2025 - December 25, 2025
  • PR Fields: Title, body, comments, commits, reviews, files changed

AI Detection Methods

  • LLM Detection: Groq Batch API with Llama 3.3 70B model analyzes PR content for AI signatures
  • Pattern Detection: 25+ regex patterns match known AI tool signatures (Copilot, Claude, Cursor, etc.)
  • Confidence Scoring: Multi-signal approach combining text analysis, commit patterns, and file changes

Metrics Definitions

  • Cycle Time: Hours from PR creation to merge
  • Review Time: Hours from PR creation to first review
  • PR Size: Total lines added + deleted
  • AI Adoption Rate: Percentage of PRs with detected AI assistance

Data Quality

  • Total PRs: 60,545
  • LLM Analyzed: 52,224 (86.3%)
  • Empty Body PRs: 6,669 (11%) - excluded from LLM analysis
  • Detection Agreement: 16.1% of PRs detected by both methods

  1. Mixed Tool Strategy Wins: Teams using bot + IDE + LLM combinations show better outcomes than single-tool teams.
  2. Review Velocity is the Consistent Win: 6/8 teams show faster review times with AI. This is the most reliable benefit.
  3. Autonomous Agents Need Oversight: Devin and Cubic produce 50-80% larger PRs. Consider more frequent checkpoints.
  4. 40-60% AI Adoption is the Sweet Spot: Very high adoption (>80%) may indicate over-reliance on bots.
