# Content Templates

Page structure templates for CTO-focused marketing content.

---

## Homepage Structure

### Section Order
1. Hero (problem + promise)
2. Pain Points (3 key problems)
3. How It Works (3 steps)
4. Features (benefit-focused)
5. Social Proof (trust builders)
6. Pricing (clear, transparent)
7. Final CTA (strong close)

### Hero Section Template
```
┌─────────────────────────────────────────────────┐
│  [Headline: Question or Problem Statement]      │
│                                                 │
│  [Subhead: 1-2 sentences explaining solution]   │
│                                                 │
│  [Primary CTA]  [Secondary CTA]                 │
│                                                 │
│  [Trust badges or quick proof point]            │
└─────────────────────────────────────────────────┘
```

**Example:**
```
Is AI actually making your team faster?

See exactly how AI tools affect your team's code—
before the board asks.

[Start Free]  [See How It Works]

No credit card required. Setup in 2 minutes.
```

### Pain Points Section Template
```
Why CTOs need this / The problem

[Pain 1: Icon + Short headline + 2 sentences]
[Pain 2: Icon + Short headline + 2 sentences]
[Pain 3: Icon + Short headline + 2 sentences]

[CTA: See How We Help]
```

### How It Works Template
```
How it works

1. [Step: Action verb + outcome]
   [1-2 sentence description]

2. [Step: Action verb + outcome]
   [1-2 sentence description]

3. [Step: Action verb + outcome]
   [1-2 sentence description]

[CTA: Get Started]
```

**Example:**
```
1. Connect your tools
   GitHub, Jira, and more. Takes 2 minutes.

2. We analyze your data
   AI patterns, team behavior, delivery metrics.

3. Get actionable insights
   Know what's working and what to fix.
```

## Comparison Page Structure

### Section Order
1. Hero (us vs them, honest framing)
2. Quick Summary (who each is for)
3. Feature Comparison Table
4. Our Honest Take (strengths/weaknesses)
5. Pricing Comparison
6. When to Choose Them
7. When to Choose Us
8. FAQ
9. Final CTA

### Hero Template
```
┌─────────────────────────────────────────────────┐
│  Tformance vs [Competitor]                      │
│                                                 │
│  An honest comparison for engineering leaders   │
│                                                 │
│  [We'll tell you where they're better too]      │
└─────────────────────────────────────────────────┘
```

### Quick Summary Template
```
The short version

[Competitor] excels at: [their core strength]
Tformance excels at: [our core strength]

Best for enterprise/workflow: [Competitor]
Best for AI impact focus: Tformance
```

| Workflow automation        | —         | ✓            |

```
Our honest take

Where [Competitor] is ahead:

### When to Choose Template

## Pricing Page Structure

### Section Order
1. Hero (simple value statement)
2. Plan Display (clear pricing)
3. What's Included (feature list)
4. Pricing Philosophy (why flat-rate)
5. FAQ
6. Final CTA

### Pricing Hero Template
```
Simple pricing. No surprises.

$499/month
Unlimited seats. Unlimited data.

[Start Free Trial]

No credit card required. Cancel anytime.
```

### Pricing Philosophy Template
```
Why flat-rate?

Per-seat pricing punishes growth.
At $30/seat, 50 engineers = $1,500/month.
At 100 engineers = $3,000/month.

We charge $499. Whether you have 20 or 200 engineers.
Your bill doesn't grow when your team does.
```

## Feature Page Structure

### Section Order
1. Hero (feature name + benefit)
2. Problem (what pain this solves)
3. Solution (how the feature works)
4. Details (deeper explanation)
5. Related Features
6. CTA

### Feature Hero Template
```
[Feature Name]

[One sentence: What it does + why it matters]

[Screenshot or visual]

[CTA: Try This Feature]
```

## Blog/Research Content Structure

### For Technical Content
```
1. Question headline
   "How do you measure AI-generated code?"

2. Acknowledge complexity
   "There's no perfect answer. Here's what we know."

3. Data/research
   [Findings, methodology, examples]

4. Practical takeaway
   "Here's what you can do today..."

5. Soft CTA
   "Tformance does this automatically."
```

### For Thought Leadership
```
1. Provocative opening
   [Challenge conventional wisdom]

2. Evidence
   [Support your claim]

3. Implications
   [What this means for CTOs]

4. Action
   [What to do about it]
```

## Email Structure

### Announcement Email
```
Subject: [News + Benefit]

[1 sentence: What's new]

[2-3 sentences: Why it matters to you]

[CTA button]

[1 sentence sign-off]
```

### Nurture Email
```
Subject: [Question or hook]

[Acknowledge a problem]

[Share insight/tip]

[Soft CTA or link to content]
```

## Section Transition Patterns

```

### Features to CTA
```
[Feature list]

[CTA]
```

### Comparison to Decision
```
[Comparison table]

Still not sure? Here's a simple test.

[Decision framework]
```

## Length Guidelines

| Content Type | Target Length |
|--------------|---------------|
| Homepage hero | 20-40 words |
| Section headline | 5-10 words |
| Feature description | 2-3 sentences |
| Comparison section | 100-200 words |
| Blog post | 800-1500 words |
| Email body | 100-200 words |
